# MVP計画

MVP駆動で段階的に機能を追加する。

## MVP1: 最小限のクロール → SQLite保存 ✅

**ゴール**: 1チャンネルのメッセージをSQLiteに保存できる

- [x] Go module初期化、基本ディレクトリ構造
- [x] Slack API接続（User Token / Bot Token両対応）
- [x] `conversations.history` でメッセージ取得（ページネーション対応）
- [x] SQLiteスキーマ作成（channels, messages テーブル）
- [x] メッセージのUPSERT保存
- [x] `crawl` サブコマンド実装

## MVP2: スレッド・差分クロール ✅

**ゴール**: スレッド返信を含む完全なメッセージ取得と差分更新

- [x] `conversations.replies` でスレッド返信を取得
- [x] 差分クロール（前回の最新タイムスタンプ以降のみ取得）
- [x] レートリミット制御（429対応、Tier 3準拠）
- [x] クロール履歴テーブル（crawl_logs）追加
- [x] スレッド差分検知（reply_count変化分のみ再取得）

## MVP3: 分析支援 ✅

**ゴール**: 保存データをSQLで分析しやすくする

- [x] `channels` サブコマンド（チャンネル一覧表示）
- [x] `stats` サブコマンド（チャンネル別メッセージ数、期間等）
- [x] `query` サブコマンド（任意SQL実行、テーブル形式出力）
- [x] ユーザー情報テーブル追加（users）
- [x] 未登録ユーザー情報の自動取得

## MVP4: 運用改善 ✅

**ゴール**: 日常的に使いやすくする

- [x] 複数チャンネル一括クロール
- [x] `--since` / `--until` オプション（期間指定）
- [x] 進捗表示（チャンネル数、メッセージ数）
- [x] `channels.yml` によるチャンネル設定管理
- [x] `crawl-search` サブコマンド（Slack検索APIでメッセージ収集）
- [ ] エクスポート機能（CSV/JSON）— 保留
