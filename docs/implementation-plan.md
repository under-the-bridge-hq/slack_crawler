# MVP計画

MVP駆動で段階的に機能を追加する。

## MVP1: 最小限のクロール → SQLite保存

**ゴール**: 1チャンネルのメッセージをSQLiteに保存できる

- [ ] Go module初期化、基本ディレクトリ構造
- [ ] Slack API接続（Bot Token認証）
- [ ] `conversations.history` でメッセージ取得（ページネーション対応）
- [ ] SQLiteスキーマ作成（channels, messages テーブル）
- [ ] メッセージのUPSERT保存
- [ ] `crawl` サブコマンド実装

## MVP2: スレッド・差分クロール

**ゴール**: スレッド返信を含む完全なメッセージ取得と差分更新

- [ ] `conversations.replies` でスレッド返信を取得
- [ ] 差分クロール（前回の最新タイムスタンプ以降のみ取得）
- [ ] レートリミット制御（429対応、Tier 3準拠）
- [ ] クロール履歴テーブル（crawl_logs）追加

## MVP3: 分析支援

**ゴール**: 保存データをSQLで分析しやすくする

- [ ] `channels` サブコマンド（チャンネル一覧表示）
- [ ] `stats` サブコマンド（チャンネル別メッセージ数、期間等）
- [ ] `query` サブコマンド（任意SQL実行、テーブル形式出力）
- [ ] ユーザー情報テーブル追加（users）

## MVP4: 運用改善

**ゴール**: 日常的に使いやすくする

- [ ] 複数チャンネル一括クロール
- [ ] `--since` / `--until` オプション（期間指定）
- [ ] 進捗表示（チャンネル数、メッセージ数）
- [ ] エクスポート機能（CSV/JSON）
